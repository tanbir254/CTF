## üß∞ Step-by-Step: Using Nmap for Vulnerability Scanning

### ‚úÖ 1. Basic Host Discovery

Find live hosts on a network:

```bash
nmap -sn 192.168.1.0/24
```

---

### ‚úÖ 2. Port Scanning

Check open ports and running services:

```bash
nmap -sS -sV -T4 192.168.1.100
```

* `-sS` = SYN scan (stealthy)
* `-sV` = Service version detection
* `-T4` = Faster timing (less stealthy)

---

### ‚úÖ 3. OS Detection and Service Fingerprinting

```bash
nmap -A 192.168.1.100
```

* `-A` enables OS detection, version detection, script scanning, and traceroute.

---

### ‚úÖ 4. Use Nmap Scripting Engine (NSE) for Vulnerabilities

Nmap comes with powerful scripts in the **Nmap Scripting Engine (NSE)**.

#### üîç Scan for known vulnerabilities:

```bash
nmap --script vuln 192.168.1.100
```

This runs all scripts in the `vuln` category.

#### üß™ Run specific PoC-related scripts:

Examples:

```bash
nmap --script=http-vuln-cve2014-3704 -p 80 192.168.1.100
```

* This scans for **Drupal SQL injection (CVE-2014-3704)**.

You can find available scripts in:

```bash
ls /usr/share/nmap/scripts/
```

Or search scripts by keyword:

```bash
ls /usr/share/nmap/scripts/ | grep ssh
```

---

### ‚úÖ 5. Output Results for Review

Save output in different formats:

```bash
nmap -sV --script vuln -oA vuln_scan 192.168.1.100
```

* `-oA` = Output in all formats: `.nmap`, `.xml`, `.gnmap`

## üîê Example: Scan a Windows Target for MS17-010 (EternalBlue)

```bash
nmap -p445 --script smb-vuln-ms17-010 192.168.1.100
```

---

## üß† Tips

* Keep your `nmap` up to date: `sudo apt update && sudo apt install nmap`
* Learn NSE scripting if you want to write custom PoCs
* Use `--script-args` to pass arguments to scripts
* Use `nmap --script-help <script-name>` to get more info
